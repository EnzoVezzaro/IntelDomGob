## 1. Instancia Pública (Instantánea, Sin Registro)

La instancia hospedada corre en modo **Público** por defecto — 20 consultas/día, sin necesidad de cuenta.

| Producto | URL |
|---------|-----|
| **Web (Demo en Vivo)** | `https://web.intel.dom.gob` |
| **Studio (Workspace)** | `https://studio.intel.dom.gob` |
| **MCP Server** | `https://mcp.intel.dom.gob/mcp` |
| **API** | `https://api.intel.dom.gob/v1` |

Solo abre **Web** o **Studio** y empieza a preguntar. O apunta cualquier cliente MCP (Claude Desktop, Cursor, nuestra CLI) al endpoint MCP.

> **Límites del nivel Público**: 20 consultas de inteligencia/día por IP. Los endpoints de datos crudos (SIL, Senado, Knowledge Graph, etc.) son **siempre gratuitos y sin medición**.

## 2. Auto-Hospedaje de Prueba (2 Minutos)

Stack completo en local — sin API key, sin registro, todas las funciones habilitadas.

```bash
git clone <repo-url> intel.dom.gob
cd intel.dom.gob
cp .env.example .env
# Edit .env: set DEFAULT_AI_API_KEY (or use DEFAULT_AI_PROVIDER=ollama for local)
./scripts/up.sh
```

| Servicio | URL Local |
|---------|-----------|
| Studio | `http://studio.localhost` |
| Web | `http://web.localhost` |
| API | `http://api.localhost/v1` |
| MCP | `http://mcp.localhost/mcp` |
| Admin | `http://admin.localhost` |
| Docs | `http://docs.localhost` |

La plataforma arranca con `REQUIRE_API_KEY=false` — todo funciona sin claves (se aplican los límites del nivel Público).

## 3. Obtener una API Key (Para Límites Mayores)

Para ir más allá del nivel Público (20/día):

1. **Auto-hospedaje**: Abre la Consola Admin (`http://admin.localhost`), inicia sesión con la clave de admin (revisa los logs o configura `INTEL_API_TOKEN` en `.env`), ve a **API Keys → Create Key**, elige un plan.
2. **Instancia pública**: Mismo flujo en `https://admin.intel.dom.gob`.

| Plan | Audiencia | Consultas/día | Streaming | Workflows |
|------|-----------|---------------|-----------|-----------|
| **Público** | Cualquiera, sin registro | 20 | ❌ | ❌ |
| **Investigador** | Periodistas, investigadores, académicos, `.gob.do` | 200 | ✅ | ❌ |
| **Pro** | Empresas, bufetes de abogados, analistas | 1,000+ (con medición) | ✅ | ✅ |
| **Institucional** | Ministerios, grandes organizaciones | Personalizado | ✅ | ✅ + SLA |

Usa la clave:

```bash
# CLI
INTEL_API_TOKEN=idg_xxx intel query "..."

# SDK
const client = createIntelClient({ baseUrl, token: "idg_xxx" });

# curl
curl -H "Authorization: Bearer idg_xxx" ...
```

## 4. Auto-Hospedaje en Producción (10 Minutos)

Cuando estés listo para producción:

```bash
# In .env
DOMAIN=intel.dom.gob
NODE_ENV=production
REQUIRE_API_KEY=true
CADDY_ACME_EMAIL=ops@yourdomain.com
# ... other production settings

./scripts/deploy.sh
```

Caddy gestiona HTTPS automáticamente vía Let's Encrypt. Solo `DOMAIN` cambia entre dev y prod — desarrolla exactamente como en producción.
