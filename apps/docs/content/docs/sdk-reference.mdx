## Instalación

```bash
npm install @intel.dom.gob/sdk
```

El SDK es exclusivamente ESM y no tiene dependencias en tiempo de ejecución más allá de `@intel.dom.gob/types`.

## Crear un Cliente

```ts
import { createClient } from "@intel.dom.gob/sdk";

const client = createClient({
  baseUrl: "http://api.localhost",   // o https://api.intel.dom.gob
  token: "your-api-key",            // opcional — omitir en modo preview
  version: "v1",                   // opcional, predeterminado "v1"
});
```

### En el Navegador

Deriva la URL de la API a partir del hostname actual (convención de subdominios):

```ts
function apiBaseUrl(): string {
  if (typeof window === "undefined") return process.env.API_BASE_URL || "http://api:4000";
  const host = window.location.host;            // studio.localhost
  const dot = host.indexOf(".");
  if (dot > 0) {
    const domain = host.slice(dot + 1);        // localhost
    return `${window.location.protocol}//api.${domain}`;  // http://api.localhost
  }
  return "/api-proxy";
}

const client = createClient({ baseUrl: apiBaseUrl() });
```

## Núcleo: Consultas de Inteligencia

```ts
// Consulta básica
const result = await client.query({
  query: "¿Cuáles son las iniciativas legislativas recientes?",
  institutions: ["senado", "camara"],   // filtro opcional
});
console.log(result.response.summary);
console.log(result.response.confidenceLevel);
console.log(result.sources.congress);      // SourceRef[]
```

### Streaming (SSE)

```ts
for await (const event of client.queryStream({ query: "..." })) {
  // event: { type: "token" | "source" | "done" | "error", data: ... }
  console.log(event);
}
```

### Chat de Seguimiento

```ts
const reply = await client.chat({
  context: result,            // IntelligenceResult anterior
  message: "¿Y sobre el senado?",
});
```

## Instituciones

```ts
const institutions = await client.listInstitutions();
// → InstitutionDescriptor[] { id, name, description, url, enabledByDefault, hasLegislative }

const found = await client.searchInstitution("senado");
```

## Datos Legislativos (SIL)

```ts
// Cámara de Diputados
await client.silCamaraIniciativas({ page: 1 });
await client.silCamaraComisiones();
await client.silCamaraSesiones();
await client.silCamaraLegislador({ query: "Juan Pérez" });

// Senado de la República
await client.silSenadoIniciativas();
await client.silSenadoBoletines();
await client.silSenadoResoluciones();
await client.silSenadoSearch({ query: "transporte" });  // búsqueda full-text, ~32k items de DSpace
await client.silSenadoSenadores();
await client.senadoNews();
```

## Grafo de Conocimiento

```ts
// Consultar el grafo
const graph = await client.graph({ entity: "Tribunal Constitucional" });

// Ingestar un IntelligenceResult
await client.graphIngest(result);
```

## Web Fetch

```ts
const page = await client.fetchUrl("https://www.senado.gob.do/...");
// → FetchedPage { url, title, text, publishedDate, dominican }
// Devuelve null en caso de 404/502.
```

## Infraestructura

```ts
await client.health();         // { status: "ok", ... }
await client.openApi();        // especificación OpenAPI (JSON)
await client.mcpTools();       // catálogo de herramientas del servidor MCP
```

## Manejo de Errores

Las respuestas que no son OK lanzan un error con el mensaje del servidor:

```ts
try {
  await client.query({ query: "..." });
} catch (e: any) {
  // e.message: "Unauthorized" | "Rate limit exceeded" | "Validation error: ..."
  console.error(e.message);
}
```

Los métodos defensivos nunca lanzan errores con payloads malformados:

- `listInstitutions()` → `[]` en caso de fallo de parseo
- `fetchUrl()` → `null` en caso de 404/502 (lanza error en 500)

## Modo Preview / Sin Autenticación

Cuando la plataforma funciona con `REQUIRE_API_KEY=false` (predeterminado en desarrollo), omite el token:

```ts
const client = createClient({ baseUrl: "http://api.localhost" });
// Funciona sin autenticación
```

En producción (`REQUIRE_API_KEY=true`), cada solicitud necesita un header `Authorization: Bearer <key>` válido — el SDK lo adjunta automáticamente cuando se proporciona un `token`.

## Tipos de TypeScript

Todos los tipos de dominio se re-exportan desde `@intel.dom.gob/sdk/types`:

```ts
import type { IntelligenceResult, SourceRef, Confidence } from "@intel.dom.gob/sdk/types";
```

Consulta [packages/types](https://github.com/EnzoVezzaro/IntelDomGob/tree/main/packages/types) para el contrato completo.
