## Prerrequisitos

- Docker y Docker Compose v2+
- Node.js 20+ (para desarrollo local sin Docker)
- Una API key de Gemini (o cualquier clave de proveedor de IA soportada)

## 1. Clonar y Configurar

```bash
git clone <repo-url> intel.dom.gob
cd intel.dom.gob
cp .env.example .env
```

Edita `.env` y configura al menos:

```bash
DEFAULT_AI_PROVIDER=gemini
DEFAULT_AI_API_KEY=your-api-key-here
DOMAIN=localhost
```

## 2. Iniciar la Plataforma

```bash
./scripts/up.sh
```

Este único comando:
- Construye todas las imágenes Docker
- Inicia todos los servicios (API, Studio, MCP, Web, Admin, Docs, SearXNG, PostgreSQL, DragonflyDB, Caddy, 5 workers)
- Ejecuta health checks
- Muestra el estado de los servicios

## 3. Acceder a la Plataforma

| Servicio | URL |
|---------|-----|
| Studio | http://studio.localhost |
| API | http://api.localhost/v1/health |
| Swagger UI | http://api.localhost/v1/docs |
| MCP Server | http://mcp.localhost/health |
| Web Client | http://web.localhost |
| Admin Console | http://admin.localhost |
| Documentación | http://docs.localhost |

## 4. Verificar el Estado

```bash
curl http://api.localhost/v1/health
```

Respuesta esperada:
```json
{
  "status": "ok",
  "timestamp": "2026-...",
  "service": "api",
  "version": "v1"
}
```

## Probar Sin una API Key (Modo Preview)

La plataforma funciona en **modo de vista previa pública** por defecto. No se requiere API key — clona, ejecuta y consulta de inmediato. Todos los clientes (Studio, Web, CLI, MCP, SDK) funcionan sin credenciales.

Cuando estés listo para más, crea API keys en la **consola de administración** y configura `REQUIRE_API_KEY=true`.

| Plan | Consultas/día | API Key | Registro |
|------|-------------|---------|--------|
| **Público** | 20 | No necesaria | No |
| **Investigador** | 200 | Requerida | Yes (`.gob.do` o manual) |
| **Pro** | 1,000+ | Requerida | Sí (pago) |

Los datos públicos crudos (instituciones, datos legislativos del SIL, lecturas del grafo de conocimiento) son **siempre gratuitos** y sin medición. Consulta la [Guía de Inicio](/docs/getting-started) para la guía completa y la tabla de niveles.

## Desarrollo Local (sin Docker)

```bash
npm install --workspaces
cd apps/api && npm run dev       # API en :4000
```
