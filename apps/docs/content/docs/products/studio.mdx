Studio es el **cliente interactivo principal** de INTEL.DOM.GOB. Es nuestro fork AGPL-3.0 de [Odysseus](https://github.com/pewdiepie-archdaemon/odysseus) — un workspace multi-agente auto-hospedado.

## Acceso

| Entorno | URL |
|---------|-----|
| Local (Docker) | `http://studio.localhost` |
| Producción | `https://studio.intel.dom.gob` |

Studio es un **stack Docker separado** (`docker/studio/v1.compose.yml`) en su propia red (`studio-net`). Ejecuta:
- `odysseus` (aplicación principal, puerto 7000)
- `studio-chromadb` (base de datos vectorial)
- `studio-searxng` (búsqueda privada)
- `studio-ntfy` (notificaciones)

## Primer arranque (Local)

```bash
# Desde la raíz del repositorio
./scripts/up.sh

# Espera las verificaciones de salud...
# Abre http://studio.localhost
```

Studio registra el servidor MCP **automáticamente en el primer arranque** mediante un hook de inicio (`scripts/studio-onboot.sh` montado en el contenedor). El endpoint MCP: `http://mcp:4100/mcp`.

## Qué incluye

| Función | Descripción |
|---------|-------------|
| **Chat** | Multi-turno, con contexto, streaming |
| **Agentes** | Planificador, Búsqueda, Recuperación, Evidencia, Validación, Respuesta |
| **Fuentes** | Web en vivo, SIL Congreso, noticias del Senado, Knowledge Graph, docs personalizados |
| **Workspaces** | Sesiones persistentes, historial, ramificación |
| **Herramientas** +40 herramientas MCP (search, fetch, SIL, Senado, KG, fetch-url, etc.) |
| **Exportación** | Markdown, PDF, JSON |

## Recorrido rápido

1. **Nuevo Chat** → escribe una pregunta → observa el pipeline de agentes (Planificar → Buscar → Recuperar → Evidencia → Respuesta)
2. **Panel de Fuentes** — cada afirmación cita evidencia (iniciativa SIL, noticia del Senado, página web, entidad del KG)
3. **Selector de Workspace** — arriba a la izquierda, crea contextos aislados por proyecto
4. **Configuración** → Modelo (Gemini, Ollama, compatible con OpenAI), Búsqueda (idioma, seguro, rango de tiempo), tema de la interfaz

## Conectar tu propio MCP

Studio solo se comunica con **un servidor MCP** (configurado en `STUDIO_MCP_URL`, por defecto `http://mcp:4100/mcp`). Para apuntar a una plataforma diferente:

```yaml
# docker/studio/v1.compose.yml
environment:
  STUDIO_MCP_URL: http://your-mcp-host:4100/mcp
```

## Instancia pública

`https://studio.intel.dom.gob` — se conecta a la plataforma pública. No se necesita cuenta para el tier **Público** (20 consultas/día). Para tiers superiores, crea una API key en Admin Console y agrégala en Studio Settings → API Key.

## Solución de problemas

| Problema | Solución |
|----------|----------|
| "MCP connection failed" | Revisa `docker compose logs mcp` — MCP debe estar saludable en el puerto 4100 |
| Studio muestra pantalla en blanco | `docker compose restart studio` — ChromaDB puede necesitar reinicio |
| Sin resultados | Verifica que `STUDIO_SEARXNG_URL` apunte a `studio-searxng:8080` (no al SearXNG de la plataforma) |
