La CLI es un **cliente MCP puro** — se conecta al servidor MCP y expone todas las herramientas mediante un prompt interactivo con marca. No hay lógica de API aquí.

## Instalación y ejecución

```bash
# Desde la raíz del repositorio
cd apps/cli
npm install
npm run dev

# O desde la raíz del monorepo
npm run dev --workspace=apps/cli
```

La CLI se conecta a `http://mcp.localhost/mcp` por defecto (Docker Compose). Para cambiar:

```bash
INTEL_MCP_URL=http://your-mcp:4100/mcp npm run dev
```

## Primer arranque

```
$ npm run dev

╭────────────────────────────────────────────────────────────╮
│  INTEL.DOM.GOB — Terminal Intelligence Client              │
│  Connected to MCP: http://mcp.localhost/mcp                │
╰────────────────────────────────────────────────────────────╯

intel> 
```

Escribe una pregunta o usa comandos.

## Comandos

| Comando | Descripción |
|---------|-------------|
| *(texto libre)* | Consulta de inteligencia multi-agente |
| `/query <texto>` | Consulta explícita |
| `/chat <texto>` | Seguimiento basado en resultado anterior |
| `/fetch <url>` | Obtener y extraer texto legible |
| `/institutions` | Listar plugins gubernamentales disponibles |
| `/tools` | Selector interactivo de herramientas MCP |
| `/help` | Mostrar ayuda |
| `/exit` | Salir |

## Modo one-shot (para scripts)

```bash
# Consulta directa
npm run dev -- -p "¿Cuáles son las últimas iniciativas sobre compras públicas?"

# Silencioso + salida JSON
npm run dev -p "query" -q -f json
```

## Modo Público / Preview

**No se necesita API key.** La CLI se autentica mediante el servidor MCP. Cuando la plataforma ejecuta con `REQUIRE_API_KEY=false` (desarrollo por defecto), todo funciona de fábrica. En producción, el servidor MCP se autentica con la API usando `INTEL_API_TOKEN` — la CLI nunca ve las claves.

## Opcional: reescritura con LLM local

La CLI puede reescribir resultados MCP estructurados en prosa fluida usando cualquier endpoint compatible con OpenAI. Configura mediante variables de entorno o prompt interactivo en la primera ejecución:

| Variable | Descripción |
|----------|-------------|
| `INTEL_LLM_BASE_URL` | Endpoint compatible con OpenAI (ej., `https://api.openai.com/v1`) |
| `INTEL_LLM_API_KEY` | API key del proveedor LLM |
| `INTEL_LLM_MODEL` | Nombre del modelo (ej., `gpt-4o-mini`) |

Cuando no está configurado, la CLI renderiza los resultados estructurados directamente (sin reescritura).

## Variables de entorno

| Variable | Valor por defecto | Descripción |
|----------|-------------------|-------------|
| `INTEL_MCP_URL` | `http://mcp.localhost/mcp` | Endpoint del servidor MCP |
| `INTEL_LLM_BASE_URL` | *(deshabilitado)* | LLM opcional para reescritura de prosa |
| `INTEL_LLM_API_KEY` | *(deshabilitado)* | API key del LLM opcional |
| `INTEL_LLM_MODEL` | *(deshabilitado)* | Modelo LLM opcional |

## Independiente (sin Docker)

Si tienes la API + MCP ejecutándose por separado:

```bash
INTEL_MCP_URL=http://localhost:4100/mcp npm run dev
```

## Arquitectura

```
CLI (apps/cli
├── src/
│   ├── index.ts           # Entrada, bucle de prompt, enrutador de comandos
│   ├── mcp-client.ts      # Transporte MCP (Streamable HTTP + SSE)
│   └── interpreter.ts     # Reescritura de prosa con LLM opcional
```

La CLI usa el **protocolo MCP** (Streamable HTTP + SSE) para llamar herramientas en el servidor MCP. Nunca llama a la API directamente.

## Solución de problemas

| Problema | Solución |
|----------|----------|
| "MCP connection failed" | Verifica `INTEL_MCP_URL`, asegúrate de que el servidor MCP esté saludable (`curl http://mcp:4100/health`) |
| "Tool not found" | Ejecuta `/tools` para ver las herramientas disponibles; la versión del servidor MCP puede diferir |
| La reescritura LLM falla | Verifica `INTEL_LLM_BASE_URL` + `INTEL_LLM_API_KEY`; déjalos sin configurar para deshabilitar |
