INTEL.DOM.GOB ships as **one private API** plus a set of **public-facing products** that talk to it. The API itself, its Swagger UI, and the OpenAPI spec are **not** public — they live behind admin authentication. This page documents the products a user or integrator actually touches.

## The products

| Product | What it is | Client surface |
|---|---|---|
| **Studio** | The primary IntelDomGob Studio workspace (AGPL-3.0 fork of Odysseus). Researchers run queries, chats, document intelligence, and workflows here. | Connects to the platform **only via the MCP server**. |
| **Web** | Lightweight, no-JS web client for quick lookups. | `@intel.dom.gob/sdk` |
| **CLI** | Command-line client for scripts and power users. | `@intel.dom.gob/sdk` |
| **MCP Server** | Model Context Protocol server exposing the platform's tools to AI assistants. A pure SDK client of the API. | `@intel.dom.gob/sdk` |
| **Admin** | Operator/Admin console for issuing API keys, viewing usage, and managing the platform. | `@intel.dom.gob/sdk` (admin scopes) |
| **SDK** | The single supported client library (`@intel.dom.gob/sdk`). Every other product is built on it. | — |

> No client imports a service or provider directly. Every request flows **Client → API → Orchestrator → Services → Providers**.

## Authentication model

Every public-facing endpoint sits behind an **API-key wall**:

- **No key → Público preview.** Runs with scopes `read, query, chat`, 20 queries/day, 10/min. No signup.
- **Valid key → that tier.** Keys are issued from the Admin console and bound to a plan; the gateway enforces the plan's scopes, rate limit, and daily quota.
- **Internal endpoints without a valid key → `401`.** Workflows, tool/plugin execution, prompt authoring, evaluation, and tenant info never fall back to preview.
- **Suspended or overdue payment → `402` hard block** on metered requests.
- **Swagger UI (`/v1/docs`) and OpenAPI (`/v1/openapi.json`) are admin-only** and are not part of the public surface.

### Tiers (Suscripciones)

| Plan | Audience | Raw data | Intelligence queries | Streaming | Workflows / Document Intelligence | Price |
|---|---|---|---|---|---|---|
| **Público** | Anyone, no signup | Unlimited | 20/day | ❌ | ❌ | Free |
| **Investigador** | Verified journalists, researchers, academics, `.gob.do` | Unlimited | 200/day | ✅ | ❌ | Free |
| **Pro** | Businesses, law firms, analysts | Unlimited | 1,000/day included, metered overage | ✅ | ✅ | Flat fee/mo |
| **Institucional** | Ministries, large orgs, integrators | Unlimited | Custom | ✅ | ✅ + dedicated tenancy, SLA, priority workers | Custom contract |

Raw official data (SIL, institution registry, knowledge-graph reads) is always free and unmetered. What is metered is the compute-intensive synthesis layer — multi-agent reasoning, document intelligence, embeddings, and workflows.
