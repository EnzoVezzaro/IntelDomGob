La API de INTEL.DOM.GOB es el punto de entrada único para todas las capacidades de la plataforma. Cada producto (Studio, Web, CLI, MCP) se comunica con la API. Tú también puedes hacerlo.

## URL base

| Entorno | URL base |
|---------|----------|
| Local (Docker) | `http://api.localhost/v1` |
| Producción | `https://api.intel.dom.gob/v1` |

Todos los endpoints están bajo `/v1`.

## Autenticación

Dos modos:

| Modo | Cómo | Caso de uso |
|------|------|-------------|
| **Preview (sin clave)** | Ninguno | Tier Público — 20 consultas/día, sin registro |
| **API Key** | `Authorization: Bearer <key>` | Tiers Investigador, Pro, Institucional — cuotas mayores, streaming, workflows |

Crea claves en **Admin Console → API Keys** (auto-hospedado) o usa la instancia pública.

```bash
# Preview (sin clave)
curl https://api.intel.dom.gob/v1/query \
  -H "Content-Type: application/json" \
  -d '{"query":"¿Qué leyes regulan las compras públicas en RD?"}'

# Con API key
curl -H "Authorization: Bearer idg_xxx" \
     -H "Content-Type: application/json" \
     https://api.intel.dom.gob/v1/query \
     -d '{"query":"¿Qué leyes regulan las compras públicas en RD?"}'
```

## Primera consulta (2 líneas)

```bash
curl -X POST https://api.intel.dom.gob/v1/query \
  -H "Content-Type: application/json" \
  -d '{"query": "¿Cuáles son las últimas iniciativas legislativas sobre compras públicas?"}'
```

Respuesta:
```json
{
  "summary": "Se identificaron 3 iniciativas recientes...",
  "detailedAnalysis": "...",
  "citations": [...],
  "validation": { "confidenceLevel": "High", ... },
  "timeline": [...],
  "evidence": [...]
}
```

## Endpoints principales

| Endpoint | Método | Descripción |
|----------|--------|-------------|
| `/query` | POST | Consulta de inteligencia completa (búsqueda → recuperación → síntesis) |
| `/chat` | POST | Pregunta de seguimiento basada en un resultado anterior |
| `/chat/stream` | POST | Chat con streaming SSE |
| `/entities/extract` | POST | Extraer Personas / Organizaciones / Leyes / Instituciones de un texto |
| `/evaluate/faithfulness` | POST | Evaluar la fidelidad de una respuesta contra el contexto |
| `/workflows` | POST | Definir y ejecutar workflows DAG de múltiples pasos |
| `/institutions` | GET | Listar plugins de datos gubernamentales registrados |
| `/health` | GET | Verificación de actividad |
| `/openapi.json` | GET | Especificación OpenAPI 3.1 completa |

## Uso del SDK (Recomendado)

```bash
npm i @intel.dom.gob/sdk
```

```ts
import { createIntelClient } from "@intel.dom.gob/sdk";

const client = createIntelClient({
  baseUrl: "https://api.intel.dom.gob",
  token: "idg_xxx",           // opcional en modo preview
  product: "my-app"           // identifica tu superficie de cliente
});

const result = await client.query({
  query: "Últimas leyes sobre transparencia en RD",
  responseLang: "es"
});

console.log(result.summary);
```

## Chat con streaming

```ts
for await (const token of client.streamChat({
  systemInstruction: "Eres un asistente legal dominicano.",
  grounding: result.detailedAnalysis,
  message: "¿Qué dice la Ley 340-06 sobre procedimientos de excepción?"
})) {
  process.stdout.write(token);
}
```

## Ejecutar la API de forma independiente

La API es un servicio Node/Express (`apps/api`). Puede ejecutarse de forma independiente:

```bash
# Desde la raíz del repositorio
cd apps/api
npm install
npm run dev      # tsx watch src/index.ts en el puerto :4000

# O con Docker (construye su propia imagen)
docker build -t intel-api -f apps/api/Dockerfile .
docker run -p 4000:4000 \
  -e DEFAULT_AI_PROVIDER=gemini \
  -e DEFAULT_AI_API_KEY=your-key \
  -e DATABASE_URL=postgres://... \
  -e REDIS_URL=redis://... \
  intel-api
```

La API requiere PostgreSQL y DragonflyDB (compatible con Redis). Consulta el docker-compose de la plataforma para el stack completo, pero la imagen de la API en sí es independiente.
