El servidor MCP expone **todas las capacidades de la plataforma** como herramientas MCP mediante el **Model Context Protocol** oficial (especificación 2025-03-26). Cualquier cliente compatible con MCP puede conectarse.

## Endpoint

| Entorno | URL |
|---------|-----|
| Local (Docker) | `http://mcp.localhost/mcp` |
| Producción | `https://mcp.intel.dom.gob/mcp` |

**Transporte:** Streamable HTTP (POST + streaming SSE) + respaldo SSE (GET). Puerto `4100`.

## Conecta cualquier cliente MCP

### Claude Desktop

```json
// ~/Library/Application Support/Claude/claude_desktop_config.json
{
  "mcpServers": {
    "intel-dom-gob": {
      "command": "npx",
      "args": ["mcp-remote", "http://mcp.localhost/mcp"]
    }
  }
}
```

### Cursor

```json
// ~/.cursor/mcp.json
{
  "mcpServers": {
    "intel-dom-gob": {
      "url": "http://mcp.localhost/mcp"
    }
  }
}
```

### Genérico (cualquier cliente MCP)

```bash
# Probar con mcp-remote
npx mcp-remote http://mcp.localhost/mcp
```

## Ejemplo de herramienta: `query`

```json
{
  "name": "query",
  "arguments": {
    "query": "¿Cuáles son las últimas iniciativas sobre compras públicas?",
    "scope": "sil",
    "responseLang": "es"
  }
}
```

Retorna `IntelligenceResult` estructurado (resumen, citas, evidencia, validación, línea de tiempo).

## Autenticación

| Modo | Cómo |
|------|------|
| **Preview** | Ninguno — funciona sin clave (tier Público) |
| **API Key** | El servidor MCP autentica con la API mediante `INTEL_API_TOKEN` (configurado en Docker Compose). El cliente nunca maneja claves. |

En producción (`REQUIRE_API_KEY=true`), el servidor MCP usa `INTEL_API_TOKEN` para llamar a la API. El cliente (Claude, Cursor, etc.) solo se conecta al endpoint MCP.

## Desarrollo local

```bash
# Iniciar el stack completo (incluye MCP)
./scripts/up.sh

# MCP disponible en http://localhost:4100/mcp
curl -X POST http://localhost:4100/mcp \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}'
```

## Transportes

| Transporte | Endpoint | Caso de uso |
|------------|----------|-------------|
| **Streamable HTTP** | `POST /mcp` | Principal (lotes JSON-RPC + streaming SSE) |
| **SSE** | `GET /mcp` | Respaldo para clientes antiguos |
| **DELETE** | `DELETE /mcp` | Terminación de sesión (no-op según especificación) |

## Versión del protocolo

- **Especificación MCP:** 2025-03-26
- **Transportes:** `jsonrpc`, `mcp-streamable-http`, `mcp-sse`
- **Endpoint:** `/mcp` (montado en la raíz)
