El cliente Web es el **sitio web público** con una demo de inteligencia en vivo. No requiere JavaScript para la experiencia base (mejora progresiva).

## Acceso

| Entorno | URL |
|---------|-----|
| Local (Docker) | `http://web.localhost` |
| Producción | `https://web.intel.dom.gob` |

## Qué hace

| Función | Descripción |
|---------|-------------|
| **Demo en vivo** | Escribe una pregunta → ve el resultado del pipeline de inteligencia completo |
| **Sin cuenta** | Funciona en el tier **Público** (20 consultas/día) sin API key |
| **Sin JS requerido** | El HTML/CSS base funciona sin JavaScript (la demo usa fetch si está disponible) |
| **Demo del SDK** | Muestra cómo llamar a la API desde el navegador |
| **Responsivo** | Mobile-first, estética terminal oscura |

## Cómo funciona

El cliente Web es un **servidor Express** (`apps/web/src/index.ts`) que sirve HTML/CSS/JS estáticos desde `public/`. Proxiea las llamadas a la API hacia el gateway interno:

```
Browser → Web (Express :4200) → API Gateway (api:4000) → Orchestrator → Agents
```

El formulario de la demo hace POST a `/query` mediante el SDK (`createClient({ baseUrl })`) y renderiza la respuesta con streaming o en buffer.

## Prueba rápida

```bash
# Local
open http://web.localhost

# O haz curl al endpoint de la demo directamente
curl -X POST http://api.localhost/v1/query \
  -H "Content-Type: application/json" \
  -d '{"query":"¿Qué es la Ley 340-06?"}'
```

## Características

| Función | Descripción |
|---------|-------------|
| **Formulario de consulta** | Input único, Enter para enviar |
| **Streaming** | Token por token si JS está habilitado |
| **Citas** | Tarjetas de evidencia expandibles |
| **Copiar/Compartir** | Copiar resultado en markdown, compartir enlace |
| **Tema** | Se adapta a la estética oscura/roja de la plataforma |
| **Accesible** | HTML semántico, contraste WCAG AA |

## Configuración

| Variable | Valor por defecto | Descripción |
|----------|-------------------|-------------|
| `API_BASE_URL` | `http://api:4000` | Gateway interno de la API |
| `WEB_PORT` | `4200` | Puerto de escucha de Express |
| `DOMAIN` | `localhost` | Para CORS / URLs absolutas |

## Desarrollo

```bash
cd apps/web
npm run dev    # tsx watch src/index.ts
```

## Producción

```bash
docker compose up -d web
# Servido por Caddy en https://web.intel.dom.gob
```

## Respaldo sin JS

Si JavaScript está deshabilitado:
- El formulario envía mediante POST estándar
- El servidor renderiza el resultado mediante plantilla
- Sin streaming, pero el resultado completo funciona
