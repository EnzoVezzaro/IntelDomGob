## Single Canonical Compose File

Development and production use the **same** `docker-compose.yml`. Only `DOMAIN` differs.

```bash
docker compose up -d        # brings up all services
docker compose ps           # health-checked services
docker compose logs -f api  # tail API logs
```

## Networking

- Only Caddy publishes ports (80/443)
- All other services use internal Docker DNS
- Services communicate by name: `api`, `searxng`, `postgres`, `dragonfly`
- Two networks: `intel` (platform) and `studio-net` (Studio isolation)

---

## Health Checks

Every container exposes `/health`, `/ready`, `/live`:

```bash
docker compose ps           # shows health status
curl http://api:4000/health  # from within the network
```

## Production Deployment

```bash
./scripts/deploy.sh
```

Internally: `git pull` → `docker compose pull` → `docker compose up -d --build` → health checks.

Suitable for self-hosting and cloud VPS without modification.
