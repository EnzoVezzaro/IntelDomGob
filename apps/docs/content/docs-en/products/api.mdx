The INTEL.DOM.GOB API is the single entry point for all platform capabilities. Every product (Studio, Web, CLI, MCP) talks to the API. You can too.

## Base URL

| Environment | Base URL |
|-------------|----------|
| Local (Docker) | `http://api.localhost/v1` |
| Production | `https://api.intel.dom.gob/v1` |

All endpoints are under `/v1`.

## Authentication

Two modes:

| Mode | How | Use case |
|------|-----|----------|
| **Preview (no key)** | None | Public / Público tier — 20 queries/day, no signup |
| **API Key** | `Authorization: Bearer <key>` | Investigador, Pro, Institucional tiers — higher quotas, streaming, workflows |

Create keys in **Admin Console → API Keys** (self-hosted) or use the public instance.

```bash
# Preview (no key)
curl https://api.intel.dom.gob/v1/query \
  -H "Content-Type: application/json" \
  -d '{"query":"¿Qué leyes regulan las compras públicas en RD?"}'

# With API key
curl -H "Authorization: Bearer idg_xxx" \
     -H "Content-Type: application/json" \
     https://api.intel.dom.gob/v1/query \
     -d '{"query":"¿Qué leyes regulan las compras públicas en RD?"}'
```

## First Query (2 lines)

```bash
curl -X POST https://api.intel.dom.gob/v1/query \
  -H "Content-Type: application/json" \
  -d '{"query": "¿Cuáles son las últimas iniciativas legislativas sobre compras públicas?"}'
```

Response:
```json
{
  "summary": "Se identificaron 3 iniciativas recientes...",
  "detailedAnalysis": "...",
  "citations": [...],
  "validation": { "confidenceLevel": "High", ... },
  "timeline": [...],
  "evidence": [...]
}
```

## Key Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/query` | POST | Full intelligence query (search → retrieve → synthesize) |
| `/chat` | POST | Follow-up question grounded in a previous result |
| `/chat/stream` | POST | SSE streaming chat |
| `/entities/extract` | POST | Extract People / Orgs / Laws / Institutions from text |
| `/evaluate/faithfulness` | POST | Evaluate answer faithfulness against context |
| `/workflows` | POST | Define & execute multi-step DAG workflows |
| `/institutions` | GET | List registered government data plugins |
| `/health` | GET | Liveness probe |
| `/openapi.json` | GET | Full OpenAPI 3.1 spec |

## Using the SDK (Recommended)

```bash
npm i @intel.dom.gob/sdk
```

```ts
import { createIntelClient } from "@intel.dom.gob/sdk";

const client = createIntelClient({
  baseUrl: "https://api.intel.dom.gob",
  token: "idg_xxx",           // optional in preview mode
  product: "my-app"           // identifies your client surface
});

const result = await client.query({
  query: "Últimas leyes sobre transparencia en RD",
  responseLang: "es"
});

console.log(result.summary);
```

## Streaming Chat

```ts
for await (const token of client.streamChat({
  systemInstruction: "Eres un asistente legal dominicano.",
  grounding: result.detailedAnalysis,
  message: "¿Qué dice la Ley 340-06 sobre procedimientos de excepción?"
})) {
  process.stdout.write(token);
}
```

## Running the API Standalone

The API is a Node/Express service (`apps/api`). It can run independently:

```bash
# From repo root
cd apps/api
npm install
npm run dev      # tsx watch src/index.ts on :4000

# Or with Docker (builds its own image)
docker build -t intel-api -f apps/api/Dockerfile .
docker run -p 4000:4000 \
  -e DEFAULT_AI_PROVIDER=gemini \
  -e DEFAULT_AI_API_KEY=your-key \
  -e DATABASE_URL=postgres://... \
  -e REDIS_URL=redis://... \
  intel-api
```

The API requires PostgreSQL and DragonflyDB (Redis-compatible). See the platform docker-compose for the full stack, but the API image itself is standalone.
